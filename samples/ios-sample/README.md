# iOS sample — run on your iPhone

A real iOS app that starts the OctetSDK and evaluates every v1
predicate against your phone's **live location**. It shows a map of
where you are, a **Generate proof & evaluate** button (each tap mints
a fresh location proof and re-runs the predicates), and a country
picker that asks the SDK `loc.isWithin(country)` for any ISO country.

The sample consumes `octetpolicy` from the repository root (`../..`).
The repo-root `Package.swift` in turn pulls the `OctetSDK` binary
xcframework from
[github.com/octetproof/octet-sdk-ios](https://github.com/octetproof/octet-sdk-ios).

## Prerequisites

- **Xcode 16 or later** (the SDK ships an `OctetSDK.xcframework` that
  needs Xcode 16 to link).
- **An Apple ID signed into Xcode** for code-signing to your iPhone.
  A free Apple ID works for personal builds; the provisioning profile
  it issues expires every 7 days.
- **An iPhone running iOS 16+.**
- **[`xcodegen`](https://github.com/yonaskolb/XcodeGen)** —
  `project.yml` is the source of truth for the Xcode project. Install
  with Homebrew: `brew install xcodegen`.
- **An OctetSDK license key.** Get one from
  [sdk.octetproof.com/signup](https://sdk.octetproof.com/signup) — a
  free trial key works for evaluation.

For deeper SDK documentation see
[docs.octetproof.com](https://docs.octetproof.com).

## 1. Generate the Xcode project

The `.xcodeproj` is **not** committed — generate it from `project.yml`:

```bash
cd samples/ios-sample
xcodegen generate
```

Re-run `xcodegen generate` any time you edit `project.yml`.

## 2. Set your signing team (one-time)

Your Apple Developer Team ID and bundle identifier live in a
**gitignored** `Local.xcconfig`:

```bash
cd samples/ios-sample/OctetPolicySample
cp Local.xcconfig.example Local.xcconfig
```

Edit `Local.xcconfig`:

```
DEVELOPMENT_TEAM = YOUR_TEAM_ID_HERE        # e.g. J8F92W9G9S
PRODUCT_BUNDLE_IDENTIFIER = com.YOUR-NAME.OctetPolicySample
```

- **Team ID**:
  [developer.apple.com/account](https://developer.apple.com/account) →
  Membership → Team ID.
- **Bundle identifier**: reverse-DNS, globally unique; use your own
  prefix.

> Why an xcconfig instead of Xcode's Signing & Capabilities UI?
> Picking your team in the UI writes the team ID *into*
> `project.pbxproj` — which is regenerated by xcodegen but would still
> leak the value through other workflows. The xcconfig keeps your
> Team ID out of every committed file.

## 3. Add your license key (one-time)

The SDK needs your license key. It lives in a **gitignored**
`LocalConfig.swift`:

```bash
cd samples/ios-sample/OctetPolicySample
cp LocalConfig.swift.example LocalConfig.swift
```

Edit `LocalConfig.swift` and set `octetLicenseKey` to your key
(`octet_live_…`). Without it the app launches but shows a "no
license key" message instead of results.

If `LocalConfig.swift` is missing entirely you'll get a *"Build input
file cannot be found"* error — that's the prompt to copy the example
file.

## 4. Build and run

### Option A — Xcode (IDE)

```bash
open samples/ios-sample/OctetPolicySample.xcodeproj
```

Xcode resolves the local package (and its remote `octet-sdk-ios`
binary dependency). Connect your iPhone, tap **Trust** on the device,
pick it in the device dropdown, and hit **⌘R**. On first launch,
grant **Location** and **Motion & Fitness** when iOS prompts (the SDK
fuses both).

If you hit *"Untrusted Developer"* on the device: on the iPhone,
**Settings → General → VPN & Device Management → your Apple ID →
Trust**, then relaunch.

### Option B — Command line

Resolve packages and build for a connected device or simulator:

```bash
cd samples/ios-sample
xcodegen generate          # if you haven't yet, or project.yml changed

# Build for an iOS Simulator:
xcodebuild build \
    -project OctetPolicySample.xcodeproj \
    -scheme OctetPolicySample \
    -destination 'generic/platform=iOS Simulator'

# Build for a connected physical device:
xcodebuild build \
    -project OctetPolicySample.xcodeproj \
    -scheme OctetPolicySample \
    -destination 'generic/platform=iOS'
```

Installing on a physical device from the command line still needs a
signed build and Apple's `xcrun devicectl` (Xcode 15+); for most
workflows it's simpler to use **Option A** for the on-device run.

## What you should see

- A **map** centered on your current location (MapKit + CoreLocation —
  the OctetSDK is a containment oracle and exposes no raw
  coordinates, so the map uses the OS location).
- A **Generate proof & evaluate** button. Tap it (give the sensors a
  few seconds first) — it mints a fresh proof and runs `isSingapore`,
  `isUS`, `isUSState([CA, NY, TX])`, and `isOfacComprehensive`
  against your live location, rendering each `PolicyResult` (match,
  reason, country/state/confidence).
- A **country picker** that calls `loc.isWithin(country)` directly
  and shows inside / outside / can't-tell.

Unless you're physically in one of the matched regions, expect
`isSingapore` / `isOfacComprehensive` to be no-match and `isUS` /
`isUSState` to reflect where you actually are. A cold start before
the sensors warm up reads back `verdictIndeterminate` — the honest
"can't tell", not a false "no"; tap again after a moment.

## How this sample is wired up

- Generated by [`xcodegen`](https://github.com/yonaskolb/XcodeGen) —
  see `project.yml`.
- Depends on the **local** SwiftPM package at `../..` (this
  repository's root). That package, in turn, depends on the public
  `OctetSDK` binary xcframework from
  [github.com/octetproof/octet-sdk-ios](https://github.com/octetproof/octet-sdk-ios).
- Single product dependency: `OctetPolicy` (the umbrella). One
  `import OctetPolicy` in `ContentView.swift` brings in every
  predicate **and** the SDK API (`Octet`, `OctetLoc`, `OctetRegion`,
  `OctetVerdict`) via the umbrella's `@_exported import`.
- Targets iOS 16+, iPhone + iPad.

## Troubleshooting

**"Build input file cannot be found: LocalConfig.swift"** — you
skipped step 3. Copy `LocalConfig.swift.example` to
`LocalConfig.swift` and fill in the key.

**"No accounts found" / "Signing for OctetPolicySample requires a
development team"** — you skipped step 2. Edit `Local.xcconfig` with
your Team ID + a unique bundle identifier.

**"license key is malformed" / "license expired"** at runtime —
trial keys from
[sdk.octetproof.com/signup](https://sdk.octetproof.com/signup) are
valid for 90 days plus a 15-day grace period. Request a new one.

**`OctetSDK` package fails to resolve** — check internet access; the
xcframework is fetched from a GitHub Release on
[octetproof/octet-sdk-ios](https://github.com/octetproof/octet-sdk-ios).
